On every case, we use a select (<app-select></app-select>) component, which is the main goal of this task. This component is mostly stateless, in order to implement many uncertain data models In future versions, if we had a structure data source, we can implement data models.

  • Version 1
    • provided data is a string array
    • the array is fixed, so you can't query the server and see performance on big payloads
  • Version 2
    • Provided data is using the model that was given
    • You can query the server to generate random data and see performance on big payloads
